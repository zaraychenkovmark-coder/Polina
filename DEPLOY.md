# Инструкция по деплою на GitHub Pages

## Шаги для деплоя:

1. **Убедитесь, что файлы скопированы:**
   - Скрипт автоматически копирует файлы из `img/` и `trap/` в `public/` при сборке
   - Или запустите вручную: `npm run copy-assets`

2. **Настройте GitHub Pages:**
   - Перейдите в Settings вашего репозитория на GitHub
   - В разделе Pages выберите:
     - Source: **GitHub Actions**
     - Branch: main (или ваша основная ветка)

3. **Закоммитьте и запушьте изменения:**
   ```bash
   git add .
   git commit -m "Настройка деплоя на GitHub Pages"
   git push origin main
   ```

4. **Автоматический деплой:**
   - После пуша в ветку `main` GitHub Actions автоматически:
     - Скопирует файлы из `img/` и `trap/` в `public/`
     - Соберет проект
     - Задеплоит на GitHub Pages
   - Проверьте статус деплоя в разделе **Actions** вашего репозитория
   - После успешного деплоя сайт будет доступен по адресу: 
     `https://<ваш-username>.github.io/<название-репозитория>`

## Важно:

- ✅ Проект использует **HashRouter**, поэтому все маршруты будут работать с `#` (например: `/#/quotes`, `/#/trap`)
- ✅ Файлы из `img/` и `trap/` автоматически копируются в `public/` при сборке
- ✅ После деплоя проверьте, что все ресурсы (изображения, аудио) загружаются корректно

## Структура URL после деплоя:

- Главная: `https://username.github.io/repo/#/`
- Цитаты: `https://username.github.io/repo/#/quotes`
- TRAP: `https://username.github.io/repo/#/trap`
